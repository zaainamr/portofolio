Overview

The Ramayana App project is a comprehensive digital transformation initiative aimed at bridging the gap between Ramayana's traditional offline sales model and the growing demand for online shopping. This project was developed as part of an Interaction Design course, focusing on creating a seamless e-commerce experience.

Starting with in-depth user research to identify pain points in the current customer journey, the project evolved into a full-scale UI/UX design and system architecture plan (UML). The goal was to provide a robust solution that retains loyal customers while attracting a tech-savvy generation.

Design Goals

Digital Transformation

Modernize the brand by shifting from an offline-only model to a competitive e-commerce presence.

Solving Pain Points

Address user frustrations with limited product access and lack of digital convenience.

System Architecture

Provide detailed UML documentation including Use Case, Sequence, and Class diagrams to ensure scalability.

Key Features

01

User-Centric Design

Interface designed based on feedback from Google Forms surveys and Miro brainstorming sessions, prioritizing ease of navigation.

02

Comprehensive Catalog

Digital catalog requiring robust categorization and search functionality to handle extensive inventory.

03

UML Documentation

Full technical breakdown (Use Case, Activity, Sequence, Class Diagrams) to bridge design with development.

04

Loyalty Integration

Seamless integration of member points and rewards to retain existing offline customer base.
